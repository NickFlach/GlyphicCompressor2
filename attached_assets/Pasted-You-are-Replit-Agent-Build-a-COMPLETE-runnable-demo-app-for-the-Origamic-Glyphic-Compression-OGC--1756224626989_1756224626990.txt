# BACKEND API (JSON; PNG/header base64)
- POST /api/generate_synthetic { n:int, density:float, seed:int } 
  -> { modelId, shapes, stats }
- POST /api/upload_model (multipart; small JSON of tensors ok)
  -> { modelId }
- POST /api/encode { modelId or tensors, params:{ partitionSize?, quant:{interiorBits,boundaryBits}, prime:{mode:'primes'|'ap'|'p2plus4q2', start?, limit?, apLen?} } 
  -> { glyphPngBase64, headerJson, stats:{origBytes, compBytes, eccBytes, sha256} }
- POST /api/decode { glyphPngBase64, headerJson }
  -> { tensors, stats:{mse, psnr, checksumOk} }
- GET  /api/inspect/:modelId 
  -> { graph:{n, edges}, partitions, boundaries, supergraph }
- GET  /api/health -> { status:'ok' }

# ALGORITHMS (concise, JS-friendly)
- Influence: if no model, synthesize block-diagonal matrix with cross-block couplers; influence тЙИ abs(A); optional random-probe Jacobian using tf.randomNormal inputs.
- Partition: тИЪn blocks; boundary if edge crosses blocks. Build supergraph between boundary nodes; intra-partition shortest paths via Dijkstra on local graph (plain JS).
- Prime schedule:
  - primes(): sieve up to limit
  - arithmeticProgression(len, startPrime, diffPrime)
  - p2plus4q2Anchors(k): choose k pairs (p,q) consecutive primes; mark high-precision pivots
- Quantization: per-partition k-means codebook (k=16/32) in JS; interiors use 3тАУ4b indices; boundaries use 6тАУ8b (or small delta residuals).
- Factorization: simple SVD mock using numeric JS (or tf.svd on small blocks); rank chosen by supergraph centrality & prime checkpoints.
- Entropy: order streams along a Hilbert-like index over partitions; deflate with zlib; store parity chunks for ECC (e.g., every Nth chunk has XOR parity or small ReedтАУSolomon).
- Glyph: canvas via pngjs; tiles map partitions via Hilbert curve; color encodes codebook family/precision; overlay arcs for boundary couplings; embed a small legend band.
